## 3. Layout Structure
1.  **Hero:**
    *   Centered Logo (ASCII or Image)
    *   Centered One-Liner
    *   Centered Badges Row
2.  **Value Prop (The "Why"):**
    *   Use blockquote `> "Quote"` for emotional hook.
    *   Use bullet points for quick scanning.
3.  **The "Prime" Stack (The "What"):**
    *   Clean Markdown Table with clear headers.
4.  **Architecture (The "How"):**
    *   Mermaid Diagram (Left-to-Right flow).
5.  **Features (The "Capabilities"):**
    *   Use `<h3>` headers with icons.
    *   Use `<details><summary>` for complex JSON/Code blocks to keep it clean.
6.  **Setup (The "Action"):**
    *   Copy-pasteable code blocks.
7.  **Footer:**
    *   Navigation table for deep links.

## 4. Formatting Rules
*   **Headers:** Sentence case for H1/H2, Title case for H3.
*   **Code:** Always specify language (`bash`, `json`, `yaml`).
*   **Emphasis:** **Bold** for key terms, *Italic* for scenarios.
*   **Spacing:** Use `<br>` if Markdown spacing feels cramped (sparingly).
